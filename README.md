# Telegram multi-agent orchestrator bot

Этот репозиторий содержит пример Telegram-бота, который в автоматическом режиме
оркестрирует работу трёх ролей ChatGPT: стратега, менеджера и джуниора. Скрипт
рассчитан на запуск в среде Google Colab, где бот можно поднимать по мере
необходимости.

## Быстрый старт

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Определите переменные окружения:
   ```bash
   export OPENAI_API_KEY="ваш_api_ключ"
   export TELEGRAM_BOT_TOKEN="токен_бота"
   ```
   В Colab сделайте то же самое через `os.environ` или `%env`.
3. Запустите скрипт:
   ```bash
   python bot.py
   ```
4. Откройте чат с ботом и отправьте задачу. Бот последовательно задействует
   стратега, менеджера и джуниора. По завершении вы получите итоговый текст и
   журнал промежуточных задач.

## Как это работает

* **Стратег (gpt-5)** — получает исходный запрос пользователя и формирует
  детальный план. План передаётся менеджеру.
* **Менеджер (gpt-4.1)** — декомпозирует план в цепочку задач, выдаёт их
  джуниору, анализирует результаты и решает, когда работа завершена.
* **Джуниор (gpt-5-nano)** — пишет контент в ответ на задачи менеджера.

Коммуникация между ролями построена через OpenAI Responses API. Менеджер и
джуниор общаются в строгом JSON-формате, что позволяет боту контролировать
рабочий процесс и предотвращать бесконечные циклы. Менеджер может повторно
ставить задачи, пока не сочтёт результат достаточным.

## Настройка

* Значения моделей (`gpt-5`, `gpt-4.1`, `gpt-5-nano`) можно изменить при
  необходимости.
* Ограничение на количество итераций менеджера задаётся константой
  `MAX_MANAGER_ITERATIONS`.
* Логи выводятся в стандартный поток, поэтому в Colab удобно следить за
  процессом через ячейку запуска.

## Безопасность

Не публикуйте токен Telegram-бота и API-ключ OpenAI. В Colab храните секреты в
переменных окружения, секретном хранилище или запрашивайте ввод у пользователя.
